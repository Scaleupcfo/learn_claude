## ① Prompt skeleton

<PromptSkeleton fields="team=Company, cycle=Q4 2026, theme=Move from product-market fit to scaling motion" template="Draft OKRs for [team] for [cycle]. Strategic theme: [theme].

Inputs:
- Top 3 things we believe matter most this quarter: [PASTE]
- What we did NOT accomplish last quarter: [PASTE]
- Constraints (cash, headcount, deadlines): [PASTE]

Produce:
- 3-5 Objectives. Each is qualitative, ambitious, and clearly worth doing.
- For each Objective, 3 Key Results. Each is measurable, time-bound, and outcome-focused (not activity-focused).
- For each KR, one sentence on how we'll know we're on track at the halfway point.

Rules:
- Reject 'launch X' as a KR. KRs are outcomes; launches are activities.
- If you can't define a metric for a KR, the KR is wrong — fix it." />

## ② How to iterate

- Force-rank: *"If we could only hit one Objective, which one? Why?"*
- Stress-test: *"What's the most ambitious version of each KR? What's the safest? Pick somewhere in between."*
- Sanity check: *"Where might these OKRs incentivize bad behavior?"*

## ③ How to run it

**Claude Cowork** for drafting. Save the final OKRs in Notion or your OKR tool with owners.

## ④ When things go wrong

- **All KRs are activities** → *"Rewrite every KR that starts with 'launch', 'build', 'ship', or 'complete' as an outcome."*
- **Too many** → *"Cut to 3 Objectives. The 4th is always the one that gets dropped."*
